# OpenClaw Integration

MedgeClaw runs on top of [OpenClaw](https://github.com/openclaw/openclaw). This guide explains how to inject MedgeClaw's identity, skills, and project context into an OpenClaw instance.

## Quick Start

```bash
cd /path/to/MedgeClaw
bash medgeclaw-init.sh
openclaw gateway restart
```

## Recovery: `medgeclaw-remind.sh`

If OpenClaw gets accidentally reset or a new session loses context:

```bash
bash /path/to/MedgeClaw/medgeclaw-remind.sh
```

This script:
- Checks if `MEDGECLAW.md`, `IDENTITY.md`, and skills config are intact
- If anything is missing, runs the full `medgeclaw-init.sh` automatically
- If everything is fine, sends a system event to remind the agent of its MedgeClaw identity

## Manual Configuration

If you prefer not to use the scripts, add this to your `~/.openclaw/openclaw.json`:

```json
{
  "skills": {
    "load": {
      "extraDirs": [
        "/path/to/MedgeClaw/skills"
      ]
    }
  }
}
```

Then copy `MEDGECLAW.md` (generated by the init script) to `~/.openclaw/workspace/`.
